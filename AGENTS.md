# Codex guide for RPT-Classic

## Project overview

RPT-Classic is a legacy multiplayer role-playing game built with BYOND and Dream
Maker (DM). The project entry point is `Roleplay Tenkaichi.dme`; its ordered
`#include` block defines the compilation unit. A `.dm` file outside that block is
not compiled, and include order can affect definitions and overrides.

## Where to look

- `Code/Items`: item types, equipment, money, and item UI.
- `Code/Mob`: player variables, lifecycle, movement, behavior, verbs, and windows.
- `Code/Skills`: skill types plus combat, buff, ki, support, and weapon systems.
- `Code/Modules`: larger systems such as world, admin, communication, saving, UI,
  health, character creation, and libraries.
- `Code/_Atom`, `Code/_Datums`, `Code/_HELPERS`: shared types, variables, procs,
  constants, and UI helpers.
- `Code/Map`: map source and build systems; included `.dmm` maps are protected.
- `Roleplay Tenkaichi.dme`: compile flags, resource directories, and include order.
- `RT.dmf`: BYOND interface definition.
- `assets/`: browser-based HTML/CSS/JavaScript UI assets.
- Editable text normally means `Code/**/*.dm`, `assets/`, project tooling, and the
  DME include list when a source-file change requires it.
- Protected areas include `Data/` (especially `Data/Saves`), databases, logs,
  `Icons/`, `Fonts/`, maps, audio, `RT.dmf`, and compiled/generated BYOND outputs.
  Do not inspect player saves or expose, copy, summarize, or overwrite player data.

## Efficient workflow

1. Start with `rg -n "term" Code "Roleplay Tenkaichi.dme" RT.dmf assets` rather
   than searching the entire repository.
2. Before changing a type or proc, search its full path and proc name. Inspect its
   parent and child types, callers, related definitions, inherited behavior, and
   every override (`..()` calls included); DM definitions are extended across files.
3. Keep a new `.dm` file focused on one system and add it to the ordered include
   block in `Roleplay Tenkaichi.dme`. A file that exists but is not included does
   not compile.
4. Run `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/check.ps1`
   after source or project-file changes.
5. If Dream Maker is installed and available on `PATH`, also compile with
   `DreamMaker "Roleplay Tenkaichi.dme"`. Treat new compiler warnings as defects.

## Editing rules

- Preserve the surrounding DM style. This is a legacy codebase with mixed casing,
  tabs, spacing, and naming. Make narrow, task-specific patches only.
- Do not perform repository-wide formatting, normalize all line endings, rename
  groups of files, or modernize legacy code outside the requested task.
- Do not rename resource paths or DM types for consistency; resources and saves may
  refer to them by name. Preserve DME include spelling, separators, casing, and
  order unless the requested change requires one deliberate new entry.
- Do not hand-edit generated/compiled files (`*.dmb`, `*.rsc`, `*.dyn.rsc`) or
  runtime logs and databases. Do not change saves, maps, icons, audio, or fonts
  unless the task specifically calls for it.
- Never change gameplay behavior or balancing while performing maintenance work.
- Never overwrite unrelated user changes or delete untracked files to clean the
  working tree. Inspect `git status --short` before and after work.
- Never run `git reset --hard`, `git clean`, or broad `checkout`/`restore` commands
  that discard work. Any destructive Git action requires explicit authorization.
- Never commit, push, merge, rebase, or modify remote branches unless explicitly
  requested.
- Never install dependencies or download tools unless required and explicitly
  approved.
- Quote paths in commands: the project name and many asset paths contain spaces.
- Do not assume a test framework exists. The repository check validates project
  structure; a Dream Maker compile is the authoritative source check.
- Some legacy `.dm` files are intentionally absent from the DME. The default check
  reports their count; use `scripts/check.ps1 -Strict` only when auditing orphans.

## Completion checklist

- The change is limited to the requested system.
- Every DME include resolves and new source files are included exactly once.
- `scripts/check.ps1` passes.
- `git diff --check`, `git status --short`, and the complete final diff were reviewed.
- Dream Maker compilation was run when the executable was available; otherwise,
  state clearly that compilation could not be performed.
- No gameplay balance, compiled output, logs, saves, database, map, unrelated user
  file, or binary asset was changed.
