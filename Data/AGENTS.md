# Instructions for `Data`

This directory contains runtime state and is deny-by-default.

- Do not read, search, summarize, copy, edit, delete, format, migrate, or commit real
  saves, backups, logs, bans, admins, keys, databases, item serials, or world state.
- Do not use files here as fixtures or examples.
- For a task explicitly about persistence, infer the schema from source and create
  synthetic data under the operating system temporary directory.
- Any operation against production data requires explicit user authorization, a backup
  plan, a dry run, and a separate narrowly scoped procedure.
